# AI 研发过程看板 - 提案

## 是什么

这是一个将 AI 辅助研发过程具象化展示的看板系统。传统的看板用于管理任务进度，而这个看板专门用于展示和跟踪一个完整的 AI 研发流水线——从需求提出、方案设计、编码实现、测试验证到部署运行的全过程。

## 为什么

### 背景

在使用 Claude Code 进行研发时，整个过程涉及多个阶段和多种 Skill 工具：
- **需求阶段**: 使用 `brainstorming`、`to-prd` 等技能
- **设计阶段**: 使用 `planner`、`architect`、`outline` 等技能
- **研发阶段**: 使用 `tdd`、`execute-plan`、`subagent-driven-development` 等技能
- **测试阶段**: 使用 `code-review`、`verification-before-completion`、`e2e-testing` 等技能
- **部署阶段**: 使用 `ship`、`land-and-deploy`、`canary` 等技能

但目前这些过程分散在对话历史中，缺乏一个统一的视图来展示整个研发流程的全貌。

### 目标

1. **可视化 AI 研发流程** - 让 AI 辅助研发的过程变得可见、可追踪
2. **展示 Skill 工具生态** - 展示 Claude Code 各类技能在研发各阶段的应用
3. **教育与演示** - 作为教学工具，帮助他人理解如何使用 AI 进行高效研发
4. **过程沉淀** - 将研发过程中的决策、设计文档、测试用例等自动沉淀到看板中

## 核心功能

### 1. 研发阶段展示

看板分为多个列，每个列代表一个研发阶段：
- 📋 **需求提出** - PRD、需求分析、可行性研究
- 🎨 **方案设计** - 架构设计、技术选型、API 设计
- 💻 **编码实现** - 功能开发、代码生成、重构
- ✅ **测试验证** - 单元测试、集成测试、E2E 测试、Code Review
- 🚀 **部署运行** - 构建、部署、监控

### 2. AI 工具调用记录

每个任务卡片记录：
- 使用的 Skill 工具
- 调用时间戳
- 输入/输出摘要
- 生成的文档/代码链接

### 3. 自动化流水线

通过 Webhook 或定时任务，自动：
- 检测代码提交并创建对应任务
- 运行测试并更新状态
- 生成构建报告并推送通知

### 4. 数据持久化

- 所有研发过程数据存入数据库
- 支持查询和统计分析
- 可导出为报告或演示文稿

## 技术实现

### 后端增强

在现有 Node.js + Express + Prisma 的基础上：
- 增加 `ai_sessions` 表记录 AI 对话会话
- 增加 `skill_calls` 表记录 Skill 工具调用
- 增加 `artifacts` 表记录生成的文档/代码
- 实现 Webhook 端点接收外部事件

### 前端增强

在现有 React + TypeScript 的基础上：
- 增加时间线视图展示研发过程
- 增加 Skill 工具使用统计
- 增加自动化流水线配置界面

## 预期价值

1. **对内价值** - 帮助自己更好地组织和追踪 AI 辅助研发工作
2. **对外展示** - 向他人展示 AI 研发的能力和流程
3. **过程改进** - 通过数据分析发现研发过程中的瓶颈和优化点
4. **知识沉淀** - 形成可复用的 AI 研发模式和最佳实践
